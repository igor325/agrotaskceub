Projeto AgroTask

Uma solução para automatizar a gestão de atividades rurais, integrando JetBov e WhatsApp para otimizar a comunicação entre gestores e a equipe de campo.

(Projeto Integrador Ildo - TCC)

O Desafio

Atualmente, gestores de fazendas que utilizam o JetBov para registrar manejos precisam comunicar as tarefas diárias (alimentação, vacinação, etc.) aos seus funcionários manualmente via WhatsApp.

  • Processo lento e suscetível a falhas.
  • Falta de confirmação de leitura e execução.
  • Comunicação descentralizada e desorganizada.
  • Dificuldade em monitorar o progresso em tempo real.

A Solução

O AgroTask é uma plataforma web que atua como o "cérebro" central, conectando as duas pontas da operação:

  • Importa dados de rebanhos, lotes e atividades da API do JetBov.
  • Automatiza o envio de tarefas via API oficial do WhatsApp (Meta).
  • Recebe confirmações de execução das tarefas dos funcionários pelo próprio WhatsApp.
  • Gera relatórios e centraliza o histórico de atividades.

Arquitetura e Integrações

O sistema é construído sobre uma arquitetura RESTful (RNF07), garantindo escalabilidade e manutenibilidade. A comunicação flui através de duas integrações-chave (RNF04).

JetBov API

Sincroniza rebanhos, lotes e atividades (RF02, RF09).

→
AgroTask

Gestão, agendamento e processamento de tarefas.

→
WhatsApp API

Envia alertas e recebe confirmações (RF04, RF05).

Banco de Dados

Todos os dados são persistidos em um banco de dados relacional PostgreSQL (RNF06), garantindo integridade e performance.

Principais Funcionalidades (Requisitos Funcionais)

Gestão de Usuários e Tarefas

Plataforma central para gestores cadastrarem usuários e criarem/atribuírem tarefas agropecuárias (alimentação, vacinação, etc.).

RF01, RF03, RF08

Integração e Sincronia JetBov

Importação automática de dados de rebanhos e lotes, e sincronização constante para manter os dados atualizados.

RF02, RF09

Notificações via WhatsApp

Envio automático de mensagens para funcionários sobre tarefas pendentes, com lembretes e status de conclusão.

RF04

Confirmação de Execução

Permite que o funcionário responda pelo próprio WhatsApp para confirmar a execução de uma tarefa, fechando o ciclo.

RF05

Relatórios e Histórico

Geração de relatórios de desempenho e produtividade, com histórico de interações para auditoria.

RF06, RF07

Exportação de Dados

Capacidade de exportar relatórios de atividades em formatos comuns, como PDF e Excel, para análise externa.

RF10

Pilares Técnicos (Requisitos Não Funcionais)

Usabilidade e Acesso

  • Interface web responsiva (mobile/desktop) (RNF01).
  • Navegação intuitiva e linguagem simples, adaptada ao público rural (RNF08).
  • Suporte a comandos de voz (detalhado no doc).

Segurança e Confiabilidade

  • Autenticação segura com JWT e criptografia SSL (RNF02).
  • Backups automáticos diários do banco de dados (RNF10).
  • Logs automáticos de falhas e eventos críticos (RNF09).

Desempenho e Escalabilidade

  • Arquitetura RESTful (RNF07).
  • Tempo de resposta médio inferior a 3s (RNF03).
  • Suporte para até 500 usuários simultâneos (RNF05).

-- WARNING: This schema is for context only and is not meant to be run.
-- Table order and constraints may not be valid for execution.

CREATE TABLE public.Activity (
  id uuid NOT NULL DEFAULT gen_random_uuid(),
  title text NOT NULL,
  description text,
  createdAt timestamp without time zone NOT NULL DEFAULT CURRENT_TIMESTAMP,
  status USER-DEFINED NOT NULL DEFAULT 'pending'::"ActivityStatus",
  isRepeating boolean DEFAULT false,
  repeatInterval integer DEFAULT 1,
  repeatUnit USER-DEFINED DEFAULT 'day'::"RepeatUnit",
  repeatEndType USER-DEFINED DEFAULT 'never'::"RepeatEndType",
  repeatEndDate date,
  repeatOccurrences integer,
  time time without time zone,
  messageTemplateId uuid,
  customMessage text,
  messageString text,
  repeatStartDate timestamp without time zone,
  scheduledDate timestamp without time zone,
  scheduledTime text,
  roles ARRAY DEFAULT '{}'::text[],
  CONSTRAINT Activity_pkey PRIMARY KEY (id),
  CONSTRAINT activity_messagetemplateid_fkey FOREIGN KEY (messageTemplateId) REFERENCES public.MessageTemplate(id)
);
CREATE TABLE public.ActivityUsers (
  activityId uuid NOT NULL,
  userId uuid NOT NULL,
  CONSTRAINT ActivityUsers_pkey PRIMARY KEY (activityId, userId),
  CONSTRAINT activityusers_activityid_fkey FOREIGN KEY (activityId) REFERENCES public.Activity(id),
  CONSTRAINT activityusers_userid_fkey FOREIGN KEY (userId) REFERENCES public.User(id)
);
CREATE TABLE public.AdminAccount (
  id uuid NOT NULL DEFAULT gen_random_uuid(),
  temporaryPassword text NOT NULL,
  createdAt timestamp without time zone NOT NULL DEFAULT CURRENT_TIMESTAMP,
  name text NOT NULL DEFAULT ''::text,
  email text NOT NULL DEFAULT ''::text UNIQUE,
  phone text NOT NULL DEFAULT ''::text,
  CONSTRAINT AdminAccount_pkey PRIMARY KEY (id)
);
CREATE TABLE public.AdminReminder (
  id uuid NOT NULL DEFAULT gen_random_uuid(),
  title text NOT NULL,
  description text,
  createdAt timestamp without time zone NOT NULL DEFAULT CURRENT_TIMESTAMP,
  isRepeating boolean DEFAULT false,
  repeatInterval integer DEFAULT 1,
  repeatUnit USER-DEFINED DEFAULT 'day'::"RepeatFrequency",
  repeatStartDate timestamp without time zone,
  repeatEndType USER-DEFINED DEFAULT 'never'::"RepeatEndType",
  repeatEndDate timestamp without time zone,
  repeatOccurrences integer,
  scheduledDate timestamp without time zone,
  scheduledTime text,
  messageString text,
  CONSTRAINT AdminReminder_pkey PRIMARY KEY (id)
);
CREATE TABLE public.MessageTemplate (
  id uuid NOT NULL DEFAULT gen_random_uuid(),
  name text NOT NULL,
  category text NOT NULL,
  templateBody text NOT NULL,
  createdAt timestamp without time zone NOT NULL DEFAULT CURRENT_TIMESTAMP,
  CONSTRAINT MessageTemplate_pkey PRIMARY KEY (id)
);
CREATE TABLE public.User (
  id uuid NOT NULL DEFAULT gen_random_uuid(),
  name text NOT NULL,
  phone text UNIQUE,
  email text UNIQUE,
  description text,
  tags ARRAY DEFAULT '{}'::text[],
  status USER-DEFINED NOT NULL DEFAULT 'active'::"UserStatus",
  createdAt timestamp without time zone NOT NULL DEFAULT CURRENT_TIMESTAMP,
  CONSTRAINT User_pkey PRIMARY KEY (id)
);
CREATE TABLE public.WorkShift (
  id uuid NOT NULL DEFAULT gen_random_uuid(),
  alertMinutesBefore integer NOT NULL DEFAULT 0,
  createdAt timestamp without time zone NOT NULL DEFAULT CURRENT_TIMESTAMP,
  startTime text NOT NULL,
  endTime text NOT NULL,
  CONSTRAINT WorkShift_pkey PRIMARY KEY (id)
);
